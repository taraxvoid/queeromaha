## Commands

```bash
bun run serve          # dev server (Astro, localhost:4321) with hot-reload
bun run serve:cms      # Decap CMS local proxy at /admin
bun run build          # build static site to dist/
bun run preview        # preview built dist/ locally

bun run lint           # check formatting (staged files only)
bun run format         # auto-fix formatting

bun test               # all tests: unit + Playwright e2e
bun run test:unit      # unit tests only (data + build)
bun run test:data      # data validation only (fastest, no build)
bun run test:e2e       # Playwright e2e only (requires built dist/)
```

Husky hooks: lint must pass on pre-commit; all tests must pass on pre-push.

## Architecture

Static site built with **Astro**, deployed to Netlify. Content is managed via **Decap CMS** (headless CMS that edits YAML files directly via git). Single layout at `src/layouts/Base.astro`.

### Content model

Directory pages live as plain YAML in `src/content/pages/*.yaml`. Each file is a category (social, spiritual, art, cafes, music, makers) with an `items` array. Simple pages (about, contact) are `src/content/pages/*.md`.

Item fields:
- `name` (required string)
- `public: false` ‚Äî hides from builds
- `description` ‚Äî rendered as markdown via `marked`
- `tags` ‚Äî array of keys from `src/data/tagMap.json`
- `links` ‚Äî array of `{ label, url }`
- `notes` ‚Äî internal notes (not displayed)

Zod schemas live in `src/content.config.ts`. The `directory` collection uses the glob loader over `*.yaml`; the `simple` collection uses glob over `*.md`.

### Routing

- `/` ‚Äî full directory, all categories shown
- `/[category]` ‚Äî directory with that category pre-filtered (e.g. `/art`, `/spiritual`)
- `/[tag]` ‚Äî directory with that tag pre-filtered (e.g. `/queer-owned`)
- `/about`, `/contact` ‚Äî simple pages

All filter routes are statically generated by `src/pages/[filter].astro`. Filter state is driven by vanilla JS in the layout (`data-initial-filter` on `<body>`, toggled by pill buttons).

### Data files

- `src/data/tagMap.json` ‚Äî canonical tag registry (emoji + label)
- `src/data/footerMessages.json` ‚Äî rotating footer messages
- `src/data/site.json` ‚Äî site-wide config (URL)

### Components

- `src/layouts/Base.astro` ‚Äî main layout (HTML shell, nav, footer, filter JS, emoji tooltips)
- `src/components/FilterBar.astro` ‚Äî category + tag filter pills
- `src/components/ItemCard.astro` ‚Äî individual `wa-card` entry

### Tests

- `test/data.test.js` ‚Äî validates YAML content structure and tag references
- `test/build.test.js` ‚Äî runs `astro build` and checks `dist/` output
- `test/e2e/site.spec.js` ‚Äî Playwright tests serving `dist/` on port 4242
