# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
bun run serve          # dev server (Astro, localhost:4321) with hot-reload
bun run serve:cms      # Decap CMS local proxy at /admin
bun run build          # build static site to dist/
bun run preview        # preview built dist/ locally

bun run lint           # check formatting (staged files only)
bun run format         # auto-fix formatting

bun test               # all tests: unit + Playwright e2e
bun run test:unit      # unit tests only (data + build)
bun run test:data      # data validation only (fastest, no build)
bun run test:e2e       # Playwright e2e only (requires built dist/)
```

Husky hooks: lint must pass on pre-commit; all tests must pass on pre-push.

## Architecture

Static site built with **Astro**, deployed to Netlify. Content is managed via **Decap CMS** (headless CMS that edits YAML files directly via git). Single layout at `src/layouts/Base.astro`.

### Content model

Directory pages live as plain YAML in `src/content/pages/*.yaml`. Each file is a category (social, spiritual, art, cafes, music, makers) with an `items` array. Simple pages (about, contact) are `src/content/pages/*.md`.

Item fields:
- `name` (required string)
- `public: false` ŌĆö hides from builds
- `description` ŌĆö rendered as markdown via `marked`
- `tags` ŌĆö array of keys from `src/data/tagMap.json`
- `links` ŌĆö array of `{ label, url }`
- `notes` ŌĆö internal notes (not displayed)

Zod schemas live in `src/content.config.ts`. The `directory` collection uses the glob loader over `*.yaml`; the `simple` collection uses glob over `*.md`.

### Routing

- `/` ŌĆö full directory, all categories shown
- `/[category]` ŌĆö directory with that category pre-filtered (e.g. `/art`, `/spiritual`)
- `/[tag]` ŌĆö directory with that tag pre-filtered (e.g. `/queer-owned`)
- `/about`, `/contact` ŌĆö simple pages

All filter routes are statically generated by `src/pages/[filter].astro`. Filter state is driven by vanilla JS in the layout (`data-initial-filter` on `<body>`, toggled by pill buttons).

### Data files

- `src/data/tagMap.json` ŌĆö canonical tag registry (emoji + label)
- `src/data/footerMessages.json` ŌĆö rotating footer messages
- `src/data/site.json` ŌĆö site-wide config (URL)

### Components

- `src/layouts/Base.astro` ŌĆö main layout (HTML shell, nav, footer, filter JS, emoji tooltips)
- `src/components/FilterBar.astro` ŌĆö category + tag filter pills
- `src/components/ItemCard.astro` ŌĆö individual `wa-card` entry

### Tests

- `test/data.test.js` ŌĆö validates YAML content structure and tag references
- `test/build.test.js` ŌĆö runs `astro build` and checks `dist/` output
- `test/e2e/site.spec.js` ŌĆö Playwright tests serving `dist/` on port 4242

Default to using Bun instead of Node.js.

- Use `bun <file>` instead of `node <file>` or `ts-node <file>`
- Use `bun test` instead of `jest` or `vitest`
- Use `bun build <file.html|file.ts|file.css>` instead of `webpack` or `esbuild`
- Use `bun install` instead of `npm install` or `yarn install` or `pnpm install`
- Use `bun run <script>` instead of `npm run <script>` or `yarn run <script>` or `pnpm run <script>`
- Use `bunx <package> <command>` instead of `npx <package> <command>`
- Bun automatically loads .env, so don't use dotenv.

## APIs

- `Bun.serve()` supports WebSockets, HTTPS, and routes. Don't use `express`.
- `bun:sqlite` for SQLite. Don't use `better-sqlite3`.
- `Bun.redis` for Redis. Don't use `ioredis`.
- `Bun.sql` for Postgres. Don't use `pg` or `postgres.js`.
- `WebSocket` is built-in. Don't use `ws`.
- Prefer `Bun.file` over `node:fs`'s readFile/writeFile
- Bun.$`ls` instead of execa.

## Frontend

Use HTML imports with `Bun.serve()`. Don't use `vite`. HTML imports fully support React, CSS, Tailwind.

Server:

```ts#index.ts
import index from "./index.html"

Bun.serve({
  routes: {
    "/": index,
    "/api/users/:id": {
      GET: (req) => {
        return new Response(JSON.stringify({ id: req.params.id }));
      },
    },
  },
  // optional websocket support
  websocket: {
    open: (ws) => {
      ws.send("Hello, world!");
    },
    message: (ws, message) => {
      ws.send(message);
    },
    close: (ws) => {
      // handle close
    }
  },
  development: {
    hmr: true,
    console: true,
  }
})
```

HTML files can import .tsx, .jsx or .js files directly and Bun's bundler will transpile & bundle automatically. `<link>` tags can point to stylesheets and Bun's CSS bundler will bundle.

```html#index.html
<html>
  <body>
    <h1>Hello, world!</h1>
    <script type="module" src="./frontend.tsx"></script>
  </body>
</html>
```

With the following `frontend.tsx`:

```tsx#frontend.tsx
import React from "react";
import { createRoot } from "react-dom/client";

// import .css files directly and it works
import './index.css';

const root = createRoot(document.body);

export default function Frontend() {
  return <h1>Hello, world!</h1>;
}

root.render(<Frontend />);
```

Then, run index.ts

```sh
bun --hot ./index.ts
```

For more information, read the Bun API docs in `node_modules/bun-types/docs/**.mdx`.
